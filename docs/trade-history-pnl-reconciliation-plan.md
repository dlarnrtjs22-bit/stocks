# 매매내역 평가손익 정합성 수정 계획

## 1. 문제 요약

- 현재 `매매내역` 탭의 보유종목 평가손익/수익률/평가금액이 키움 앱과 일치하지 않는다.
- 사용자 기준 실제 키움 계좌 표시값:
  - 현대차: `+763`
  - 두산에너빌리티: `+6,578`
- 현재 앱은 같은 시점에 더 큰 값 또는 다른 값으로 표시된다.

## 2. 현재 확인된 사실

### 2-1. 계좌 평가 TR 기준값

- `kt00018`
  - 정규장 기준 계좌평가를 내려준다.
  - 예시:
    - 현대차 `evltv_prft = -4,228`
    - 두산에너빌리티 `evltv_prft = -1,805`
- `kt00005`
  - 수수료/세금이 반영된 평가금액/평가손익 구조를 확인할 수 있다.
  - `evlt_amt`, `evltv_prft`, `pl_rt`가 키움 계산식에 더 가깝다.

### 2-2. 장 종료 후 실시간 시세

- 장 종료 후 현재 앱은 `ka10001 + NXT 종목코드(_NX)` 시세를 사용하고 있다.
- 이 시세 자체는 장후 변동을 반영할 수 있으나,
  - 현재 앱 계산은 한동안 `단순 현재가 - 평단가` 방식이 섞였고,
  - 키움 앱은 `평가손익 = 시세 반영 + 수수료/세금 포함` 구조일 가능성이 높다.

### 2-3. 사용자 제시 숫자와의 정합성

- 사용자 제공 값은 `NXT 실시간 시세`에
  - 매수수수료
  - 예상 매도수수료
  - 예상 세금
  이 포함된 키움식 평가손익과 매우 유사하다.
- 즉, 단순 `현재가 * 수량 - 매입금액`으로는 맞출 수 없고,
  `키움 계좌 평가 방식`을 그대로 따라가야 한다.

## 3. 핵심 가설

### 가설 A

- 장 마감 후 키움 앱은 `정규장 계좌평가 TR 값`을 그대로 쓰지 않고,
  `장후(NXT) 현재가`를 기준으로 `평가손익 재계산`을 하고 있을 가능성이 높다.

### 가설 B

- 재계산 시 단순 가격차가 아니라 아래 항목이 반영된다.
  - 매수수수료
  - 예상 매도수수료
  - 예상 세금
  - 키움의 반올림/절사 규칙

## 4. 수정 목표

- `매매내역` 탭의 보유종목 수치가 키움 앱과 최대한 동일하게 보이도록 수정한다.
- 목표 항목:
  - 현재가
  - 평가금액
  - 평가손익
  - 수익률
  - 상단 요약 카드의 총 평가금액 / 총 평가손익 / 추정자산

## 5. 개발 계획

### 1단계. 계산 기준 고정

- 장중/장후를 구분하는 기준을 명확히 한다.
- 정규장 중:
  - KRX 기준 계좌평가 사용 여부 검토
- 장후:
  - `effective_venue()` 기준으로 `NXT` 현재가 사용

### 2단계. 키움식 평가손익 계산 함수 분리

- 공통 계산 함수 추가:
  - 입력:
    - 보유수량
    - 평단가
    - 현재가
    - 매수수수료
    - 예상 매도수수료
    - 예상 세금
  - 출력:
    - 평가금액
    - 평가손익
    - 수익률
- 수수료/세금 계산은
  - `kt00018`/`kt00005`에서 확인된 필드 구조를 우선 사용하고,
  - 없는 경우에는 키움 앱과 가장 가까운 계산식으로 추정한다.

### 3단계. 보유종목별 계산에 계좌 API 필드 활용

- 종목별로 다음 필드를 적극 활용한다.
  - `pur_amt`
  - `pur_cmsn`
  - `sell_cmsn`
  - `tax`
  - `evlt_amt`
  - `evltv_prft`
- 정규장 기준 TR 구조를 바탕으로,
  장후에는 `현재가`만 NXT 시세로 치환한 뒤 나머지 비용 계산을 동일 규칙으로 재적용한다.

### 4단계. 상단 요약 재합산

- 보유종목별 확정 계산 결과를 합산해서 상단 카드 값을 만든다.
- 즉, 상단 카드가 별도 TR 숫자를 그대로 복사하는 구조가 아니라
  `표 하단 합계`와 일관되게 보이도록 통일한다.

### 5단계. 표시 문구 보강

- 상단 상태 문구에 현재 기준을 명시한다.
  - 예시:
    - `시세 NXT 기준`
    - `평가손익 키움식 비용 반영`
- 사용자가 숫자 기준을 바로 알 수 있게 한다.

## 6. 검증 계획

### 검증 1. 사용자 실제 계좌와 대조

- 현대차
  - 키움 앱 평가손익과 앱 평가손익 비교
- 두산에너빌리티
  - 키움 앱 평가손익과 앱 평가손익 비교

### 검증 2. 합계 검증

- 종목별 평가손익 합산 = 상단 총 평가손익
- 종목별 평가금액 합산 + 현금성 자산 = 추정자산

### 검증 3. 시간대 검증

- 정규장 중
- 15:40 이후 NXT 시간대
- 수동 새로고침
- 자동 새로고침(30초/1분)
